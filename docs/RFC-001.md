# RFC-001: Cybernetics Agent — Architecture & Interface Design

> Status: **Released v0.1.0**  
> Author: Hermes Agent  
> Date: 2026-05-12  
> Version: 0.1.0

---

## 1. 设计目标

构建一个**框架无关**的控制论 Agent 增强层，让任何 Python Agent 都能通过声明式配置或编程式接入，获得钱学森《工程控制论》七大原则的能力。

**核心约束**:
- 零外部依赖（stdlib only）
- 框架无关（不绑定任何 LLM 框架）
- 声明式配置优先（JSON 为主，YAML 为辅）
- 渐进式接入（可以只用一个模块，也可以全量启用）
- 线程安全（内置 threading locks）

**设计原则**:
1. **如无必要，勿增实体**（奥卡姆剑刀）— 不引入不必要的依赖、不创建不必要的抽象层、不添加不必要的配置项。每行代码都必须有存在的理由。
2. **大把拆小，组合为工作流** — 模块之间零依赖，单独可用、单独可测。复杂功能不是一个巨大的类，而是多个小模块的组合。

## 3. 核心抽象接口

### 3.1 CyberneticsConfig — 配置中心

```python
from dataclasses import dataclass, field
from typing import Dict, List, Optional, Any
from copy import deepcopy

_DEFAULTS: Dict[str, Any] = {
    "version": "1.0",
    "project_name": "unnamed-project",
    "feedback_loop": {"enabled": True, "mode": "automatic", ...},
    "stability": {"enabled": True, "retry": {"max_retries": 3}, ...},
    # ... 其他模块默认配置
}

@dataclass
class CyberneticsConfig:
    """声明式配置对象。对应 cybernetics.json。"""

    _data: Dict[str, Any] = field(default_factory=lambda: deepcopy(_DEFAULTS))

    @classmethod
    def from_json(cls, path: str) -> "CyberneticsConfig":
        data = json.load(open(path))
        return cls(**{k: data.get(k, _DEFAULTS.get(k)) for k in _DEFAULTS})

    def to_dict(self) -> Dict[str, Any]:
        return dict(self._data)

    def get(self, key: str, default: Any = None) -> Any:
        return self._data.get(key, default)

    @property
    def project_name(self) -> str:
        return self._data.get("project_name", "unnamed-project")

    @property
    def feedback_loop(self) -> Dict[str, Any]:
        return self._data.get("feedback_loop", {})
    # ... 其他属性
```

**设计说明**：配置系统采用单一 dataclass 包装通用 dict，避免 21 个嵌套 dataclass 的维护负担。
核心模块均通过 `config: Dict[str, Any]` 接收配置，保持简洁。

### 3.2 ICyberneticsModule — 模块基类

每个原则对应一个模块，必须实现以下接口：

```python
from abc import ABC, abstractmethod
from typing import Any, Dict, Optional

class ICyberneticsModule(ABC):
    """
    所有控制论模块的基类。
    模块是“装饰器”——它们不替代业务逻辑，只是在关键节点插入控制逻辑。
    """

    name: str  # 模块名，如 "feedback_loop"
    enabled: bool = True

    def __init__(self, config: Dict[str, Any], ctx: "CyberneticsContext") -> None:
        self.config = config
        self.ctx = ctx

    @abstractmethod
    def on_event(self, event: "CyberneticsEvent") -> Optional["CyberneticsEvent"]:
        """
        处理事件。返回 None 表示事件被消耗。
        返回修改后的事件表示继续传播。
        """
        ...

    @abstractmethod
    def get_status(self) -> Dict[str, Any]:
        """返回模块当前状态，用于审计报告。"""
        ...
```

### 3.3 CyberneticsEvent — 事件对象

```python
from dataclasses import dataclass
from typing import Dict, Any, Optional
from enum import Enum
import time

class EventType(Enum):
    # Agent 生命周期
    AGENT_START = "agent_start"
    AGENT_END = "agent_end"
    # 工具调用
    TOOL_CALL = "tool_call"
    TOOL_RESULT = "tool_result"
    TOOL_ERROR = "tool_error"
    # LLM 交互
    LLM_REQUEST = "llm_request"
    LLM_RESPONSE = "llm_response"
    LLM_ERROR = "llm_error"
    # 用户交互
    USER_INPUT = "user_input"
    USER_FEEDBACK = "user_feedback"
    # 系统
    STAGE_TRANSITION = "stage_transition"
    ERROR = "error"
    RECOVERY = "recovery"
    DEGRADATION = "degradation"
    # 自定义
    CUSTOM = "custom"

@dataclass
class CyberneticsEvent:
    event_type: EventType
    timestamp: float  # time.time()
    session_id: str
    payload: Dict[str, Any]  # 事件载荷
    metadata: Dict[str, Any] = field(default_factory=dict)
    event_id: str = field(default_factory=lambda: uuid.uuid4().hex)
```

### 3.4 CyberneticsContext — 运行时上下文

```python
class CyberneticsContext:
    """
    运行时上下文。所有模块通过它访问共享状态。
    """

    def __init__(self, config: CyberneticsConfig) -> None:
        self.config = config
        self.event_bus = EventBus()
        self.state_manager = StateManager(config.storage)
        self.metrics = MetricsCollector()
        self.modules: Dict[str, ICyberneticsModule] = {}

    def register_module(self, module: ICyberneticsModule) -> None:
        """注册模块。"""
        self.modules[module.name] = module
        self.event_bus.subscribe(module)

    def emit(self, event: CyberneticsEvent) -> None:
        """发射事件，所有订阅了该事件类型的模块会收到。"""
        self.event_bus.emit(event)

    def get_module(self, name: str) -> Optional[ICyberneticsModule]:
        """获取模块实例。"""
        return self.modules.get(name)
```

## 5. 适配层设计

### 5.1 三种接入模式

| 模式 | 适用场景 | 侵入性 | 灵活度 |
|------|---------|--------|--------|
| **Hook** | 框架已有 hook 机制（Hermes、LangChain Callback） | 低 | 高 |
| **Middleware** | 框架支持中间件模式（FastAPI、CrewAI Task） | 中 | 中 |
| **Wrapper** | 框架没有 hook 或 middleware（纯 Python、AutoGen） | 高 | 低 |

### 5.2 通用适配接口

```python
class BaseAdapter(ABC):
    """
    所有框架适配器的基类。
    """

    def __init__(self, ctx: CyberneticsContext) -> None:
        self.ctx = ctx

    @abstractmethod
    def install(self, target: Any) -> None:
        """将控制论层安装到目标框架上。"""
        ...

    @abstractmethod
    def uninstall(self) -> None:
        """卸载控制论层。"""
        ...

    def emit(self, event_type: EventType, payload: Dict[str, Any]) -> None:
        """便捷方法：创建并发射事件。"""
        event = CyberneticsEvent(
            event_type=event_type,
            timestamp=time.time(),
            session_id=self._get_session_id(),
            payload=payload,
        )
        self.ctx.emit(event)
```

### 5.3 Hermes 适配示例

```python
class HermesAdapter(BaseAdapter):
    """
    Hermes Agent 插件适配。利用 Hermes 的 plugin hook 机制。
    """

    def install(self, target: Any) -> None:
        # target 是 Hermes 的 PluginContext
        target.register_hook("post_tool_call", self._on_tool_call)
        target.register_hook("post_llm_response", self._on_llm_response)
        target.register_hook("on_error", self._on_error)

    def _on_tool_call(self, tool_name: str, result: Any, session_id: str, **kwargs) -> None:
        self.emit(EventType.TOOL_RESULT, {
            "tool_name": tool_name,
            "result": result,
            "success": not isinstance(result, Exception),
        })
```

### 5.4 LangChain 适配示例

```python
class LangChainAdapter(BaseAdapter, BaseCallbackHandler):
    """
    LangChain Callback 适配。继承 LangChain 的 BaseCallbackHandler。
    """

    def install(self, target: Any) -> None:
        # target 是 LangChain 的 LLM 或 Agent
        target.callbacks = [self]

    def on_tool_end(self, output: str, **kwargs: Any) -> Any:
        self.emit(EventType.TOOL_RESULT, {"output": output})

    def on_llm_error(self, error: BaseException, **kwargs: Any) -> Any:
        self.emit(EventType.LLM_ERROR, {"error": str(error)})
```

## 7. 开发规范

### 7.1 代码风格
- 中文注释、中文错误消息
- Python 3.9+ 兼容
- 纯标准库，无外部依赖
- 类型注解尽量完整
- 测试放在 tests/ 目录，不在模块内联冒烟测试

### 7.2 测试金字塔
```
单元测试（pytest，模块级）
    ↓ 通过后
集成测试（pytest，端到端）
    ↓ 通过后
验收（与具体框架的兼容性）
```

### 7.3 版本管理
- 主分支 `main`
- 功能分支 `feat/<name>`
- 修复分支 `fix/<name>`
- PR 合并前必须通过 CI
