# v0.6.0 设计书

> 目标: 事件持久化 | 原则: 如无必要勿增实体

---

## 核心

所有事件、指标、告警历史写入 SQLite，进程重启不丢失。

---

## 模块

```
runtime/event_store.py   # EventStore 类
```

---

## 表结构

```sql
events           — timestamp, event_type, payload, session_id
metrics          — timestamp, metric_name, metric_value, labels
alerts           — timestamp, rule_name, severity, message, metric_name, metric_value, labels
```

---

## 集成点

1. **MetricsCollector** — emit_event 时同步写入 events 表
2. **AlertManager** — evaluate 触发时同步写入 alerts 表
3. **Dashboard** — 新增 /api/history/* 端点
4. **Dashboard 前端** — 增加历史趋势图表

---

## API

```
GET /api/history/events?from=unix&to=unix&type=&limit=100
GET /api/history/metrics?metric=name&from=&to=&limit=100
GET /api/history/alerts?from=&to=&severity=&limit=100
```
