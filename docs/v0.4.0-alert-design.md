# v0.4.0 Alert & Notify 设计书

> 状态: 草案 | 目标版本: 0.4.0 | 设计原则: ① 如无必要，勿增实体 ② 大把拆小

---

## 1. 设计目标

让 Cybernetics Agent 具有「观测 → 判断 → 行动」的完整闭环。

- **观测**: v0.3.0 已完成（Metrics + Dashboard）
- **判断**: 本版本核心（告警规则引擎）
- **行动**: 本版核心（多渠道通知）

对外是独立开源项目，不依赖 Hermes 或任何特定个人工作流。用户可以自由组合模块。

---

## 2. 模块拆分

遵循「大把拆小」原则，每个模块单独可用、单独可测。

```
src/cybernetics_agent/
├── alert/
│   ├── __init__.py           # 暴露核心 API
│   ├── core.py               # AlertEvent, AlertRule 基础类
│   ├── manager.py            # AlertManager 协调器
│   ├── rules.py              # 内置规则引擎
│   ├── channels/
│   │   ├── __init__.py
│   │   ├── base.py         # AlertChannel 抽象基类
│   │   ├── stdout.py       # 标准输出（默认，零依赖）
│   │   ├── webhook.py      # 通用 HTTP Webhook
│   │   ├── feishu.py       # 飞书
│   │   ├── discord.py      # Discord
│   │   ├── dingtalk.py     # 钉钉
│   │   ├── slack.py        # Slack
│   │   └── email.py        # SMTP 邮件
│   └── ...
```

### 2.1 模块依赖关系

```
alert/core.py          ← 无依赖
alert/rules.py         ← core.py
alert/channels/base.py ← core.py
alert/channels/*.py    ← base.py
alert/manager.py       ← core.py + rules.py + channels/*.py
```

**重要**: 每个 channel 模块之间** 零依赖**。用户只导入自己需要的 channel。

## 3. 接口设计

### 3.1 AlertEvent

```python
@dataclass
class AlertEvent:
    """告警事件。"""
    rule_name: str
    severity: str       # "info" | "warning" | "error" | "critical"
    message: str
    metric_name: str | None
    metric_value: float | None
    labels: dict[str, str]
    timestamp: float
```

### 3.2 AlertChannel 抽象

```python
class AlertChannel(ABC):
    """告警渠道基类。"""
    name: str = ""

    @abstractmethod
    def send(self, event: AlertEvent) -> bool:
        """发送告警。返回是否成功。"""

    def health_check(self) -> bool:
        """健康检查。默认返回 True。"""
        return True
```

### 3.3 AlertRule 抽象

```python
class AlertRule(ABC):
    """告警规则基类。"""
    name: str = ""
    severity: str = "warning"
    channels: list[str] = []    # 指定响应渠道名称

    @abstractmethod
    def evaluate(self, metrics: MetricsCollector) -> AlertEvent | None:
        """评估是否触发告警。返回事件或 None。"""
```

### 3.4 内置规则引擎

```python
# 阈值规则
ThresholdRule(
    metric="tool_error_rate",
    operator=">",      # 支持 >, <, >=, <=, ==, !=
    threshold=0.3,
    duration=60,       # 持续 60 秒才触发
)

# 频率限制
RateRule(
    max_alerts=5,
    window=300,        # 5 分钟内最多 5 条
)

# 静默期
SilenceRule(
    duration=1800,     # 触发后静默 30 分钟
)
```

## 5. 使用示例

### 5.1 CLI

```bash
# 测试所有渠道
cybernetix alert test

# 测试单个渠道
cybernetix alert test --channel feishu

# 查看当前告警规则状态
cybernetix alert status

# 手动触发一条测试告警
cybernetix alert fire --message "测试告警" --severity info
```

### 5.2 代码

```python
from cybernetics_agent.alert import AlertManager, ThresholdRule
from cybernetics_agent.alert.channels import StdoutChannel, FeishuChannel

manager = AlertManager(ctx)
manager.add_rule(ThresholdRule(
    metric="tool_error_rate",
    operator=">",
    threshold=0.3,
))
manager.register_channel("stdout", StdoutChannel())
manager.register_channel("feishu", FeishuChannel(webhook_url="..."))

# 每隔 30 秒评估一次
manager.start_evaluator(interval=30)
```

## 6. 渠道实现说明

| 渠道 | 依赖 | 配置字段 | 说明 |
|---|---|---|---|
| stdout | 无 | 无 | 默认启用，零依赖 |
| webhook | urllib | url, headers | 通用 HTTP POST |
| feishu | urllib | webhook_url, secret | 支持签名验证 |
| discord | urllib | webhook_url | Discord Webhook |
| dingtalk | urllib | webhook_url, secret | 支持签名验证 |
| slack | urllib | webhook_url | Slack Incoming Webhook |
| email | smtplib | smtp_host, port, user, password, to | SMTP |

**原则1**: 所有渠道均使用标准库（urllib / smtplib），无外部依赖。
**原则2**: 每个渠道单独测试，互不影响。
